# 전체보기 랭킹 페이지 (`src/render.js` → `pageHTML`)

케러셀 카드의 **[전체보기]** 버튼이 여는 모바일 웹 페이지입니다.
해당 카테고리의 **전체 펀드 목록**을 자산유형 필터·정렬 토글과 함께 보여줍니다.

## 렌더링 방식

서버가 랭킹 데이터를 **JSON 으로 HTML 에 내장**해 단일 문서로 응답하고
([src/render.js:117](../../src/render.js#L117)), 필터·정렬은 브라우저에서 처리합니다.

```
GET /page/:cat → 데이터 조회 → pageHTML(cfg, data) → HTML 1개 응답
                                  ↑ 이후 조작은 추가 API 호출 없음
```

- 외부 JS 프레임워크 없음 — 순수 인라인 스크립트(IIFE)
- 데이터 삽입 시 `safeJson()` 이 `<` 를 이스케이프해 `</script>` 조기 종료를 방지 ([src/render.js:4](../../src/render.js#L4))
- 서버 렌더 문자열은 `esc()` 로 HTML 이스케이프 ([src/render.js:7](../../src/render.js#L7))

## 화면 구성

| 영역 | 내용 |
|------|------|
| 헤더 | 오렌지 그라디언트 · 브랜드 라벨 · `cardTitle` · 기준일 |
| 컨트롤 (sticky) | 좌: `자산유형` 라벨 / 우: 정렬 토글 · 아래: 자산유형 칩 목록 |
| 목록 | `N개 펀드 · <정렬 라벨>` 카운트 + 펀드 카드 목록 |
| 푸터 | 투자 유의 문구 (과거 실적 · 투자설명서 확인 안내) |

펀드 카드 한 장의 구성:

- **순위**: 필터·정렬 후의 표시 순번. 원본 순위와 다르면 아래에 `전체 N` 표기 ([src/render.js:156](../../src/render.js#L156))
- **정보**: 펀드명(2줄 클램프) · 자산유형/설명 태그 · 위험등급 배지
- **수익률**: 현재 정렬 기준 수익률(대) + 나머지 기간 수익률(소)

## 자산유형 필터 (칩)

- 고정 순서 우선: `전체 · 국내주식 · 해외주식 · 국내혼합 · 해외혼합 · 국내채권 · 해외채권 · 대안자산`
- **데이터에 실제로 등장하는 유형만** 노출하고, 목록에 없는 새 유형이 나타나면 뒤에 자동 추가
  ([src/render.js:127](../../src/render.js#L127)) → API 에 새 자산유형이 생겨도 코드 수정 불필요
- 기본 선택: `전체`

## 정렬 토글

버튼 구성은 카테고리의 `mode` 에 따라 자동으로 달라집니다 ([categories.md](./categories.md) 참고).

| 정렬 키 | 동작 | 카운트 라벨 |
|---------|------|-------------|
| `rank` | 원본 순위 오름차순 (판매/관심 인기순) | `판매 많은순` / `관심 많은순` (`countWord`) |
| `m3` | 3개월 수익률 내림차순 | `3개월 수익률 높은순` |
| `m6` | 6개월 수익률 내림차순 | `6개월 수익률 높은순` |

수익률 표시 규칙 ([src/render.js:152](../../src/render.js#L152)):

- `rank` 정렬일 때 — 큰 숫자는 3개월, 보조는 6개월
- `m3`/`m6` 정렬일 때 — 큰 숫자는 정렬 기준 기간, 보조는 나머지 기간

## 스타일 규칙

- 최대 폭 `520px` 중앙 정렬 (모바일 우선, 데스크톱에서도 카드형 유지)
- 색상: 오렌지 `#F58220` / 블루 `#043B72`, 수익률 양수=오렌지 · 음수=블루 `#0d5bb5`
- 서체: Spoqa Han Sans → 맑은고딕 폴백
- `env(safe-area-inset-*)` 로 노치 기기 대응, 카드 등장 애니메이션(`pop`)은 순번당 18ms 지연(최대 300ms)
- 위험등급 배지 CSS(`RISK_BADGE_CSS`)는 섬네일과 공유 → 두 화면의 색상 체계가 항상 일치
